This project was developed as part of my Compiler Construction coursework in November 2025.
I designed and implemented a custom **Urdu-based programming language** with its own syntax, keywords, and execution model.

The system includes core compiler phases and execution support:

- **Lexical Analysis (Scanner):** tokenizes Urdu keywords, identifiers, operators, punctuation, and numbers.
- **Parsing + Interpretation:** parses arithmetic expressions, variable declarations, assignments, conditions, loops, functions, and return statements.
- **Control Flow Support:** includes `if/else`, `while`, and `for` style behavior with Urdu-language constructs.
- **Function Handling:** supports function definitions, parameter passing, local scope, and return values.
- **Scope Management:** maintains call stack and variable memory for global/local execution contexts.
- **Error Reporting:** reports syntax and runtime errors with token/line context.

In addition to language execution, I built a desktop IDE using **Tkinter** to improve usability:

- code editor with undo/redo and right-click actions
- quick-insert sidebar for Urdu language templates
- integrated output console
- one-click run workflow for compile-and-execute behavior

This project gave me practical experience implementing language tooling end-to-end: from token design and grammar-driven parsing to runtime semantics and developer tooling.
